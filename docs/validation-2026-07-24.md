# 층-1 좌석 상태 모형 검증 보고서: 구간합 역산과 관측 귀속 규약

작성일: 2026-07-24 · 상태: 검증 기록 (표본 소량, 확정 결론 아님)
관련 문서: `docs/boarding-model.md`(v1 명세), `docs/boarding-model-v2.md`(v2 설계 방향)
대상: 경기도 직행좌석버스 3330·1650, 2026-07-20~24 평일 수집분

## 0. 요약

이번 주에 확립된 것은 다섯 가지다.

1. 층-1(노선 좌석 상태) rolling-origin 백테스트 하네스를 구축했고, 현행 프로파일 전파가 단순 기준선(좌석 유지, v1 만석빈도)을 구간합 지표에서 이긴다는 것을 확인했다.
2. 균등 배분(1/n)을 제거한 **구간합 역산** 모델을 구현했고, 합성 검증(§11.1)에서 알려진 수요 스파이크를 복원함을 확인했다.
3. 첫 필드 실측(범계역)이 API 잔여석과 현실의 일치를 확인했고(Phase 0 부분 통과), 좌석 차분만으로는 불가능한 분해(하차 인원 특정)를 실제로 수행했다.
4. 실데이터 역산이 수요 스파이크를 실제 위치보다 **한 정류장 하류**에 배치하는 계통 오프셋을 드러냈다. 이는 API 잔여석이 정류장 "도착(승차 반영 전)" 상태라는 신호이며, v2 §4.2가 Phase 0의 선결 질문으로 꼽은 항목에 대한 데이터 기반 답이다.
5. 귀속 규약(도착/출발)은 **구간합 관측만으로는 원리적으로 식별 불가능**함을 확인했다. 규약을 가른 유일한 증거는 필드 기록이었다. 산발적 필드 실측의 가치가 여기서 확립된다.

균등 배분 제거의 효과는 지표에 따라 다르다. 구간합 MAE·Brier는 동급이고(평가가 합 단위라 뭉갬이 상쇄되는 구조), 정류장 단위 수요 지도와 핫스팟 하류 예측에서는 결정적으로 개선된다. 제품이 답할 질문("이 정류장에 도착하면 몇 석 남는가")은 후자에 속한다.

## 1. 배경과 질문

현행 집계(`build-data.ts` → `shared/profile.ts`)는 v1 방식이다: 10분 간격 관측쌍의 좌석 차분 Δ를 통과 정류장 n개에 균등(1/n) 배분한다. v2 §4.3은 이 방식이 "데이터가 말하지 않은 공간 패턴을 인위적으로 생성한다"고 지적했다.

2026-07-24 아침 범계역 필드 실측이 이 지적을 실물로 만들었다. 프로파일이 기대한 범계역 순수요는 대당 3.1석이었는데 실측은 30석, 약 10배 차이였다. 질문은 다음과 같다. **균등 배분을 제거하면 얼마나 개선되는가?**

## 2. 필드 실측 기록 (2026-07-24 금, 3330 상행, 롯데백화점.범계역 seq55)

| 시각 | 관측 |
|---|---|
| 08:37 | 정류장 도착, 줄 합류. 앞 대기 7명 (Q=7) |
| 08:47 | 버스 도착. 앞 7명 중 2명만 탑승(거의 만석), 앞 5명 잔류, 뒤로 약 20명 합류 |
| 08:50 | 2층버스 도착, 잔여석 54. 기록자 포함 30명 탑승, 잔여석 24. 하차 0명. 착석 성공 |

같은 시간대 수집 스냅샷과 대조하면 두 버스 모두 특정된다(차량 식별자는 데이터 경계 원칙에 따라 기재하지 않는다). 탑승한 2층버스의 API 궤적은 `08:43 seq53 = 55석 → 08:53 seq56 = 19석`으로, 실측(도착 54석 → 처리 후 24석, 이후 문화의거리에서 추가 승차)과 맞물린다. 08:47의 만석 버스도 `08:43 seq53 = 13석 → 08:53 seq58 = 0석` 궤적과 부합한다.

이 한 건이 주는 것이 두 가지 있다.

- **Phase 0 부분 통과.** API 잔여석이 현장 좌석 수와 일치함을 확인했다 (이 케이스에 한함).
- **식별 불가능성의 실증적 해소.** API만으로는 순수요 `N = 54−24 = 30 = B−A`에서 탑승 B와 하차 A를 가를 수 없다(§1.1). 필드 라벨 B=30이 들어오자 A=0으로 특정됐다. 좌석 관측이 못 하는 분해를 필드 기록 한 숫자가 해낸다.

## 3. 구간합 역산 모델 (균등 배분 제거)

### 3.1 수식

현행(균등 배분):

```text
관측쌍 i:  구간 (a,b],  n = b−a,  Δᵢ = s(a) − s(b)
각 정류장 k ∈ (a,b] 에 표본값 Δᵢ/n 을 가중치 1/n 로 기록
μ_k = Σᵢ (1/nᵢ)(Δᵢ/nᵢ) / Σᵢ (1/nᵢ)          ← 배분을 가정한다
```

교체(구간합 역산, span-sum deconvolution):

```text
잠재:  정류장별 순수요 N_k ~ (μ_k, σ²_k) 독립   (셀 = 정류장 × 30분버킷 × 평일/주말)
관측:  Δᵢ = Σ_{k∈Kᵢ} N_k                       ← 구간은 합만 준다. 배분을 가정하지 않는다

평균:  min_μ  Σᵢ ( Δᵢ − Σ_{k∈Kᵢ} μ_k )²  +  λ Σ_k μ_k²        (ridge, λ=1)
       좌표하강 닫힌 갱신:  μ_k ← Σ_{i∋k} ( Δᵢ − Σ_{j∈Kᵢ∖{k}} μ_j ) / (n_k + λ)

분산:  eᵢ = Δᵢ − Σ μ̂_k 에 대해  E[eᵢ²] = Σ_{k∈Kᵢ} σ²_k
       σ²_k 도 같은 좌표하강으로 비음수 복원

예측:  기존과 동일 (좌석 분포에 N(μ_k, σ_k) 이산화를 정류장 순서로 DP 적용)
```

식별의 원천은 운행마다 어긋나는 구간 경계의 겹침이다. 겹침이 어긋나지 않으면(추가 정보 없음) ridge에 의해 균등 배분과 같은 해로 퇴화한다. 균등 배분은 이 모델의 무정보 특수해다.

### 3.2 합성 검증 (§11.1 게이트)

정류장 10에만 수요 25(나머지 1)를 심고 2~4정류장 구간합만 관측시킨 합성 운행 60개에서:

| | 정류장 9 | **10 (스파이크)** | 11 |
|---|---|---|---|
| 진짜 | 1 | **25** | 1 |
| 균등 배분 | 6.0 | 9.4 | 5.9 |
| 구간합 역산 | 2.0 | **23.7** | 1.5 |

균등 배분은 스파이크를 이웃으로 흘려 뭉개고, 역산은 복원한다. 게이트 통과.

## 4. 백테스트 (rolling-origin, 평일 4폴드, 관측쌍 6,480건)

프로파일은 test일 이전 날짜로만 학습한다(§11.2). 과제는 같은 운행의 관측쌍 (a, s(a)) → (b, s(b))에서 s(b)를 예측하는 것이다(자기지도 정답).

| 지표 | naive(좌석 유지) | 균등(현행) | 역산 |
|---|---|---|---|
| MAE 전체 | 7.04 | **4.14** | 4.19 |
| MAE 1–2 정류장 | 2.78 | **2.34** | 2.49 |
| MAE 3–5 | 7.84 | **4.55** | 4.65 |
| MAE 6+ | 7.39 | 4.11 | **3.93** |
| 만석 Brier | 0.0824 | **0.0434** | 0.0462 |
| 예측구간 포함률 (목표 80%) | — | 72.1% | **74.9%** |

참고: v1 만석빈도(full-frequency) Brier 0.0670, 보수 기준선 0.0878. 만석확률 보정은 두 프로파일 모두 전 구간 과소예측이 잔존한다(예: 예측 0.68~0.72 구간의 실제 만석률 0.82~0.87).

**해석.** 구간합 지표에서 역산은 균등과 동급이다. 이는 실패가 아니라 구조다: 평가 대상이 "구간의 합"인데 균등 배분은 합을 보존하도록 뭉개므로, 합 단위 채점에서는 벌점을 받지 않는다. 뭉갬의 대가는 정류장 단위 질문에서만 드러난다(7절).

## 5. 발견 1: +1 정류장 귀속 오프셋

실데이터 역산(평일 08:30 버킷, 7/20~23 학습)이 범계역 스파이크를 찾아냈으나 **한 정류장 하류에 배치했다**: μ(55 범계역) = −0.4, μ(56 문화의거리) = +11.3. 필드 기록으로 스파이크 위치가 55임은 확정이므로, 이는 솔버 오류가 아니라 관측 좌표계의 계통 밀림이다.

풀이는 다음과 같다. API `remainingSeats`가 정류장 **도착(승차 반영 전)** 상태라면, 구간 (a,b]의 좌석 차분은 실제로는 정류장 a~b−1의 승차를 담는다. 그런데 귀속은 a+1~b에 하므로 모든 수요가 +1 이동한다.

합성 매트릭스로 이 기제를 확증했다. 데이터 의미(출발/도착 상태) × 귀속 규약의 2×2 전부에서, 규약이 일치하면 스파이크가 제자리(10)에 복원되고 어긋나면 정확히 ±1 밀린다:

| 데이터 의미 | 귀속 | 스파이크 위치 |
|---|---|---|
| 출발 상태 | (a,b] | 10 ✓ |
| 출발 상태 | [a,b−1] | 9 (−1 밀림) |
| 도착 상태 | (a,b] | **11 (+1 밀림 — 실데이터 현상 재현)** |
| 도착 상태 | [a,b−1] | 10 ✓ |

이로써 v2 §4.2가 Phase 0의 선결 질문으로 꼽은 "표시 잔여석이 정류장 처리 전인가 후인가"에 대한 데이터 기반 답이 나왔다: **도착(처리 전) 상태에 가깝다.** v2 §4.7이 "후보 정류장 m−1까지만 전파하라"고 미리 써 둔 것과 정합한다.

## 6. 발견 2: 귀속 규약은 구간합만으로 식별 불가능하다

귀속을 [a,b−1]로 바꾼 변형(균등·역산 각각)을 백테스트에 추가한 결과, **모든 지표가 기존 귀속과 소수점 셋째 자리까지 동일**했다. 이는 우연이 아니라 수학적 필연이다. 모든 구간의 커버 집합이 통째로 한 칸 이동하는 좌표 재라벨링이므로, 구간 (a→b) 예측에 적용되는 셀 값의 다중집합이 두 규약에서 완전히 같아진다.

따라서 10분 구간합 데이터만으로는 규약을 원리적으로 가릴 수 없다. 이번에 규약을 판별한 유일한 증거는 **필드 기록**(스파이크의 실제 위치)이었다. 30초 라이브 관측(같은 정류장의 승차 전·후를 모두 포착)도 판별력을 가질 것으로 예상되나 아직 축적 전이다. 산발적이라도 필드 실측이 모델 검증에서 대체 불가능한 역할을 한다는 것이 이번 주의 핵심 교훈이다.

결정적 비교 — "문화의거리(56) 도착 시 잔여석" 예측 (7/24 실측 버스, seq53 도착 55석에서 출발, 실측 **24석**):

| 모델 | 예측 |

## 8. 한계

1. 평일 4일 표본이며 마지막 날은 오전 위주 부분 데이터다. 모든 수치는 방향성이지 결론이 아니다.
2. 도착 규약 판별은 **단일 필드 사례**에 근거한다. 추가 실측으로 재확인해야 한다. 특히 08:53 관측(문화의거리 19석, 도착 상태라면 24석이어야 함)은 정차 중 갱신이 섞이는 **혼합 규약**의 가능성을 시사한다. 순수 이분법이 아닐 수 있다.
3. 구간의 시간 버킷을 끝점 관측 시각으로 일괄 귀속하는 관례는 두 모델 모두에 남아 있다.
4. 역산의 분산 추정은 잔차 재역산 근사로, 과소 추정 가능성이 있다(포함률 74.9% < 목표 80%).
5. 운행 재구성(가명 차량 관측의 시간순 분할)이 옳다는 가정 위의 평가다.
6. 만석 검열 구간은 여전히 표본에서 제외된다. 혼잡 과소예측의 잔존 원인 중 하나로 추정된다(v2 §1.4).

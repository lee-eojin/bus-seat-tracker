# 실시간 좌석 상태와 사용자 대기열을 분리한 광역버스 탑승 지점 추천 모델 (v2)

작성일: 2026-07-21
상태: 검증 중인 설계 방향. Phase 0(API 의미 검증)과 단순 기준선 대비 백테스트를 통과하기 전까지 최종 명세가 아니다.
관계 문서: v1 명세는 `docs/boarding-model.md`에 그대로 보존하며, 백테스트의 비교 기준선(11.4절 기준선 2·4번의 원형)으로 사용한다.
대상 노선: 경기도 직행좌석버스 3330, 1650

## 0. 결론

기존 명세의 문제의식과 실시간 좌석 전파 아이디어는 유지한다. 그러나 잔여석 차분만으로 개인의 탑승 성공확률을 복원할 수 있다는 전제는 폐기한다. 잔여석 차분이 보여주는 것은 정류장 전체의 순변화이고, 개인에게 필요한 값은 그 사람이 줄에 들어선 시점에 이미 앞에 있던 사람 수이기 때문이다.

재설계 모델은 다음 두 문제를 분리한다.

1. **노선 좌석 상태 모형**: 현재 차량이 후보 정류장에 도착하기 직전 몇 석을 가지고 있을지 예측한다.
2. **사용자 경쟁 수요 모형**: 사용자가 후보 정류장에 도착했을 때 앞에 있는 대기자와 해당 정류장의 하차 인원이 만들어내는 좌석 압력을 예측한다.

두 모형은 날씨, 배차 간격, 일별 혼잡도 같은 공통 요인을 공유한다. 후보별 결과는 서로 독립적으로 계산하지 않고 동일한 시나리오 안에서 공동으로 생성한다. 마지막으로 사용자가 후보 정류장까지 제때 갈 확률과 이동·실패 비용을 포함해 행동을 추천한다.

이 구조로 바꿔도 데이터가 없는 것을 알아낼 수 있게 되는 것은 아니다. 후보 정류장의 대기열·하차 실측이 부족하면 개인 탑승확률 대신 보수적 하한만 제시한다. 이 제한은 모델의 약점이 아니라, 관측만으로 식별할 수 없는 값을 확률처럼 보이게 만들지 않기 위한 안전장치다.

### 0.1 문서 지위와 채택 조건

이 문서는 설계 방향으로 채택되었으나 검증 완료된 모델이 아니다. 채택 근거와 그 한계는 부록 A에 기록한다. 세 층을 구분한다.

1. v1의 개인 탑승확률 식별 결함은 수학적으로 확정된다 (1.1절, 부록 A.1).
2. 그 결함이 실제 운영 자료에서 무시하기 어렵다는 것은 3일간의 API 자료로 강하게 확인되었다 (부록 A.2, A.3).
3. 편향의 정확한 크기와 v2의 예측 우월성은 아직 입증되지 않았다 (부록 A.4).

따라서 각 구성요소는 Phase 0 통과 후, 11.4절의 단순 기준선보다 나은 성능을 백테스트로 보인 경우에만 구현을 유지한다.

### 0.2 제품 모드 선언

v1에는 두 개의 서로 다른 의사결정 문제가 섞여 있었다. v2는 이를 분리해 선언한다.

- **계획 모드**: 출발 전에 어느 정류장으로 갈지 정하는 문제. 특정 차량이 아니라 시간대 패턴에 근거한다. 이 문서의 범위 밖이며, 필요 시 별도 기능으로 설계한다.
- **실시간 모드**: 지금 보이는 특정 차량을 타기 위해 어느 정류장으로 이동할지 정하는 문제. 제시간 도착 사건 `F`가 필수다. 이 문서(v2.1)는 실시간 모드를 다룬다.

현재 제품 비전(실시간 차량 위치 표시와 그 차량에 대한 조건부 예측)은 실시간 모드에 해당한다.

## 1. 재설계가 필요한 논리적 근거

### 1.1 잔여석 차분만으로 개인 탑승 성공을 식별할 수 없다

차량이 정류장에 도착하기 직전의 잔여석을 `S⁻`, 하차 인원을 `A`, 전체 탑승 수요를 `D`, 정류장 처리 후 잔여석을 `S⁺`라 하자. 만석 경계에 닿지 않았다면 다음이 성립한다.

```text
S⁺ = S⁻ + A − D
S⁻ − S⁺ = D − A
```

잔여석 차분은 `D−A`만 보여준다. `D`와 `A`는 따로 보여주지 않는다.

예를 들어 `S⁻=3`, `S⁺=1`인 관측은 아래 두 상황에서 똑같이 발생한다.

```text
상황 1: A=0, D=2
상황 2: A=5, D=7
```

두 상황 모두 `D−A=2`이므로 API 관측은 동일하다. 하지만 사용자 앞에 4명이 있다면 상황 1에서는 탈 수 없고 상황 2에서는 탈 수 있다. 같은 잔여석 관측이 서로 다른 개인 탑승 결과와 양립한다.

따라서 잔여석만으로 `A`, 전체 수요 `D`, 사용자 앞 대기자 `Q`를 동시에 복원하는 것은 구조적으로 불가능하다. 표본을 더 모으면 분산은 줄일 수 있지만, 관측되지 않는 변수를 구분할 수 없는 식별 문제 자체는 사라지지 않는다.

### 1.2 정류장 전체 수요와 사용자 앞 수요는 다른 변수다

사용자가 버스보다 5분 먼저 도착했다면 그 뒤에 온 사람은 정류장 전체 탑승 인원에는 포함되지만 사용자보다 뒤에 선다. 정상적인 선착순 승차라면 그 사람들은 사용자의 탑승 성공을 방해하지 않는다.

따라서 다음 두 값을 구분해야 한다.

- `D`: 버스가 정류장을 처리하는 동안 탑승하려 한 전체 수요
- `Q(τᵘ)`: 사용자가 시각 `τᵘ`에 줄에 합류했을 때 이미 앞에 있던 사람 수

기존 차분은 첫 번째 값의 순변화에 관한 정보만 준다. 개인 성공식에는 두 번째 값이 필요하다.

`D`를 `Q`의 대용으로 쓰면 그 효과만 보면 경쟁을 과대평가하는 보수적 편향이다. 다만 v1 전체의 순편향 방향은 이것만으로 정해지지 않는다. 만석 표본 제외와 숨은 검열은 수요를 낮게, `D`의 대용 사용은 경쟁을 높게 추정하므로 서로 반대 방향이며, 순효과는 자료 없이 확정할 수 없다. 확정할 수 있는 것은 후보별 편향 크기가 달라 서열이 뒤집힐 수 있다는 점이다.

### 1.3 검열은 끝점의 0 여부만으로 판정할 수 없다

10분 간격 관측 사이에 여러 정류장이 포함되면 시작과 끝 잔여석이 모두 양수여도 중간에 만석이 됐다가 하차로 좌석이 다시 생겼을 수 있다. 이 경우 끝점 차분은 중간 정류장의 충족되지 못한 수요를 전혀 보여주지 않는다.

따라서 `s₁=0 또는 s₂=0`일 때만 검열로 보는 규칙은 충분하지 않다. 장구간 관측은 정류장별 독립 표본으로 나누지 않고 전체 상태 전이에 대한 제약으로 사용해야 한다.

실증 근거는 부록 A.3에 있다. 양수 관측 사이에 0석 관측이 끼는 삼중 패턴이 직접 관측되었으므로 만석 후 회복을 허용하는 상태 전이는 필수다. 다만 현재 10분 해상도에서 측정된 숨은 검열의 크기는 약 1% 수준으로 작다.

### 1.4 만석 표본을 버리면 혼잡한 날이 선택적으로 사라진다

만석일수록 수요 정보가 중요하지만 기존 방식은 만석이 포함된 관측쌍을 평균과 일일 배수 계산에서 제외한다. 그러면 수요가 높은 날일수록 학습 데이터에서 더 많이 빠진다. 결과적으로 평균 수요와 일일 혼잡 효과가 모두 낮게 추정될 가능성이 있다.

재설계에서는 만석 관측을 제거하지 않는다. 정확한 값이 아니라 `잠재 순수요가 이 임계값 이상`이라는 꼬리 제약으로 우도에 포함한다.

실증 근거는 부록 A.2에 있다. 검열이 피크 시간대에 집중되는 선택적 결측은 확인되었고, 같은 셀 내부의 편향 크기는 아직 측정되지 않았다.

### 1.5 부호가 있는 순수요에 단일 비율 배수를 적용하기 어렵다

정류장 순수요 `D−A`는 음수가 될 수 있다. 탑승이 많은 정류장과 하차가 많은 정류장을 합하면 서로 상쇄된다. 이 합을 분모로 사용한 비율은 분모가 0에 가까워지거나 음수가 될 수 있다.

또한 비가 오는 날 탑승 수요와 하차 수요가 같은 비율로 증가한다고 볼 근거도 없다. 따라서 노선 전체에 하나의 곱셈 배수 `λ`를 적용하는 대신, 부호 있는 순수요에 적용할 수 있는 가산형 공유 일별 효과와 정류장별 반응계수를 사용한다.

실증 근거는 부록 A.2에 있다. 유효 셀의 절반 이상이 음수 순수요라는 구조적 조건은 확인되었고, 실제 계산된 λ가 불안정했는지는 아직 계산하지 않았다.

### 1.6 후보별 좌석과 수요는 독립이 아니다

수요가 많은 날에는 상류에서 좌석이 빨리 줄고 후보 정류장의 대기열도 길어진다. 두 현상은 같은 원인에 의해 동시에 악화된다. 좌석 분포와 대기 수요 분포를 독립적으로 뽑아 결합하면 동시에 나빠지는 시나리오가 과소표현되어 실패 위험이 낮게 나올 수 있다.

재설계에서는 같은 일별 혼잡 상태, 날씨, 배차 간격을 좌석 모형과 대기열 모형이 공유한다.

### 1.7 추천은 정류장 속성이 아니라 사용자의 행동 결과다

상류 정류장의 탑승확률이 높아도 사용자가 버스보다 늦게 도착하면 그 선택은 실패다. 따라서 실제 사건은 `그 정류장에 좌석이 있는가`가 아니라 다음 두 사건의 교집합이다.

1. 사용자가 버스보다 먼저 후보 정류장에 도착한다.
2. 사용자 차례까지 이용 가능한 좌석이 남는다.

이동 시간과 버스 ETA를 확률변수로 포함하지 않으면 추천 문제를 풀었다고 할 수 없다.

## 2. 재설계 목표

의사결정 시각을 `t₀`, 후보 정류장 집합을 `C`, 대상 운행을 `r`이라 한다. 후보 정류장 `m`으로 지금 이동하는 행동을 `a_m`이라 한다.

재설계의 1차 목표는 다음 조건부 확률을 추정하는 것이다.

```text
P(Yᵣ,ₘ = 1 | t₀까지 관측된 정보, 행동 aₘ)
```

`Yᵣ,ₘ=1`은 사용자가 후보 정류장 `m`으로 이동해 대상 차량 `r`에 실제로 탑승한 사건이다.

추천의 최종 목표는 확률이 가장 큰 정류장을 기계적으로 고르는 것이 아니라 사용자의 기대손실을 최소화하는 것이다.

```text
a* = argminₐ E[L(a) | 현재 정보]
```

손실에는 이동 시간, 추가 대기, 탑승 실패, 목적지 지각 위험을 포함한다. 실패 비용이 큰 사용자와 걷기 비용이 큰 사용자가 같은 추천을 받을 이유가 없기 때문이다.

### 2.1 v2의 적용 범위

- v2.1은 현재 화면에서 선택한 대상 차량 또는 가장 먼저 도착할 수 있는 차량 한 대를 평가한다.
- 해당 차량을 놓친 뒤 다음 차량까지 이어지는 대기열 변화는 실패 비용에 포함하되 직접 시뮬레이션하지 않는다.
- 여러 차량과 미탑승자 이월 대기열을 연속으로 모형화하는 기능은 v2.2로 분리한다.

이 범위 제한은 첫 구현에서 식별하기 어려운 이월 대기열까지 한 번에 추정하는 것을 막는다.

## 3. 시간 기준과 변수 정의

모든 잔여석 값은 정류장 처리 전과 처리 후를 구분한다.

| 기호 | 의미 |
|---|---|
| `Cᵣ` | 대상 차량의 좌석 정원 |
| `S⁻ᵣ,ₖ` | 차량 `r`이 정류장 `k`에 도착해 하차가 시작되기 직전의 잔여석 |
| `Aᵣ,ₖ` | 정류장 `k`의 하차 인원 |
| `Dᵣ,ₖ` | 해당 차량에 탑승하려 한 전체 잠재 수요 |
| `Bᵣ,ₖ` | 실제 탑승 인원, `min(D, S⁻+A)` |
| `Uᵣ,ₖ` | 타지 못한 수요, `max(D−S⁻−A, 0)` |
| `S⁺ᵣ,ₖ` | 승하차가 끝난 직후의 잔여석 |
| `Nᵣ,ₖ` | 전체 순좌석 압력, `D−A` |
| `τᵘₘ` | 사용자가 후보 정류장 `m`에 도착하는 시각 |
| `Qᵣ,ₘ(τᵘₘ)` | 사용자가 도착했을 때 이미 앞에 있던 대기자 수 |
| `Hᵣ,ₘ` | 개인 순경쟁 압력, `Q(τᵘ)−A` |
| `Tᵇᵣ,ₘ` | 버스가 후보 정류장의 승차 처리를 시작하는 시각 |

정류장 상태 전이는 다음과 같다.

```text
Bᵣ,ₖ = min(Dᵣ,ₖ, S⁻ᵣ,ₖ + Aᵣ,ₖ)
Uᵣ,ₖ = max(Dᵣ,ₖ − S⁻ᵣ,ₖ − Aᵣ,ₖ, 0)
S⁺ᵣ,ₖ = S⁻ᵣ,ₖ + Aᵣ,ₖ − Bᵣ,ₖ
        = clip(S⁻ᵣ,ₖ − Nᵣ,ₖ, 0, Cᵣ)
```

사용자가 제시간에 도착한 사건을 다음과 같이 둔다.

```text
Fᵣ,ₘ = 1[τᵘₘ + δₘ ≤ Tᵇᵣ,ₘ]
```

`δₘ`은 길 건너기, 정류장 진입, GPS·ETA 지연을 흡수하는 안전 여유다.

선착순 승차가 지켜진다는 조건에서 개인 탑승 성공은 다음과 같다.

```text
Yᵣ,ₘ = Fᵣ,ₘ · 1[S⁻ᵣ,ₘ + Aᵣ,ₘ − Qᵣ,ₘ(τᵘₘ) > 0]
      = Fᵣ,ₘ · 1[S⁻ᵣ,ₘ − Hᵣ,ₘ > 0]
```

이 식은 노선 전체 순수요 `N`과 사용자 개인 순경쟁 압력 `H`를 명시적으로 분리한다.

## 4. 층 1: 노선 좌석 상태 모형

### 4.1 관측의 역할을 분리한다

기저 수집과 라이브 폴링을 같은 종류의 표본처럼 취급하지 않는다.

| 관측 | 주 역할 | 제한 |
|---|---|---|
| 10분 기저 수집 | 시간대·요일 전체의 대표성, 운행 위치와 만석 구간의 장기 분포 | 여러 정류장이 섞여 정류장별 귀속이 약함 |
| 30초 라이브 폴링 | 인접 정류장 전이와 상태 변화의 정밀 관측 | 사용자가 페이지를 연 시간에 치우치는 선택 편향 |
| 사용자 실측 | 대기열·하차·실제 탑승 결과의 기준값 | 표본이 적고 선택한 정류장에만 존재 |

30초 자료는 정밀하지만 무작위 표본이 아니다. 사용자가 평소보다 불안하거나 특정 시간에만 화면을 열면 표본 조건이 달라진다. 따라서 수집 방식과 페이지 활성 조건을 함께 저장하고, 기저 수집의 대표성을 라이브 자료가 대체한다고 간주하지 않는다.

### 4.2 API 관측 의미를 먼저 확정한다

구현 전에 반드시 다음을 실측으로 확인한다.

- 표시 잔여석이 현재 정류장 처리 전인지 처리 후인지
- 차량 위치 `seq`가 도착, 정차, 출발 중 어느 상태를 뜻하는지
- API timestamp와 실제 상태 사이의 지연 분포
- 동일 차량 식별자가 회차·방향 전환에서 어떻게 바뀌는지
- 차량별 좌석 정원을 확보할 수 있는지

이 의미가 확정되지 않으면 `S⁻`와 `S⁺`를 매핑할 수 없어 이후 수식 전체가 흔들린다. 매핑 신뢰도가 낮은 관측은 학습에서 제거하거나 관측 오차 모형에 포함한다.

### 4.3 균등 구간 배분을 제거한다

장구간 차분 `Δ`를 통과한 정류장 수로 나누어 각 정류장의 표본으로 만들지 않는다. 균등 배분은 데이터가 말하지 않은 공간 패턴을 인위적으로 생성한다.

대신 시작 상태에서 끝 상태까지 정류장별 잠재 순좌석 압력 `N`을 차례로 적용한다.

```text
S⁺ᵣ,ⱼ = transition(
  S⁺ᵣ,ᵢ,
  Nᵣ,ᵢ₊₁, …, Nᵣ,ⱼ
)
```

중간에 잔여석이 0에 닿았다가 다시 증가하는 경로도 이 전이 안에서 허용한다. 장구간 관측은 어느 한 정류장의 값으로 귀속되지 않고 가능한 전체 경로들의 우도를 제한한다. 계산은 동적계획법, 순차 몬테카를로 또는 입자 필터로 근사할 수 있다.

### 4.4 검열을 상태 전이 안에서 처리한다

정확한 단일 정류장 전이이고 관측 오차가 없다고 가정하면 다음 정보를 얻는다.

```text
0 < S⁺ < C  → N = S⁻ − S⁺
S⁺ = 0      → N ≥ S⁻
S⁺ = C      → N ≤ S⁻ − C
```

만석 관측은 `N`의 정확한 값은 주지 않지만 하한은 준다. 이 표본을 폐기하지 않고 꼬리확률로 학습한다. 장구간에서는 각 중간 상태를 숨은 상태로 두어 같은 원칙을 반복 적용한다.

이를 고전적인 독립 우측검열로 단순화하지 않는다. 검열 임계값 `S⁻`도 상류 혼잡도의 영향을 받기 때문이다. 상류 상태와 현재 순수요를 같은 상태공간 안에서 조건화함으로써 정보성 검열을 완화한다.

### 4.5 일일 배수 대신 계층적 공유 효과를 둔다

부호가 있는 `N`에는 가산형 모형을 사용한다.

```text
Nᵣ,ₖ = μₖ,ᵦ,ᵍ
       + aₖu_day
       + βᵀx_day
       + γ log(headwayᵣ,ₖ)
       + vᵣ
       + εᵣ,ₖ
```

- `μₖ,ᵦ,ᵍ`: 정류장·도착 시간 버킷·요일군의 기준 순좌석 압력
- `u_day`: 그날 노선 전체에 공유되는 숨은 혼잡 효과
- `aₖ`: 같은 일별 충격에 대한 정류장별 반응 정도
- `x_day`: 강수, 공휴일, 방학 등 관측 가능한 날짜 변수
- `headway`: 앞차와의 시간 간격
- `vᵣ`: 운행 단위 효과
- `ε`: 정류장별 잔차

`N`은 음수가 될 수 있으므로 잔차 분포도 양수 전용 로그정규분포가 아니라 정수형 경험분포, 이산 Student-t 등 양·음 값을 모두 허용하는 분포를 사용한다.

정류장별 표본이 적을 때는 `μ`와 `a`를 인접 시간대·요일군·노선 평균 쪽으로 부분 풀링한다. 완전히 합치지 않기 때문에 특정 정류장의 국지적 패턴을 보존하면서도 작은 표본의 극단값을 줄일 수 있다.

### 4.6 실시간 갱신은 비율 계산이 아니라 사후 갱신으로 한다

아침 시작 시 `u_day`는 날씨·요일 조건의 과거 분포를 가진다. 당일의 정밀 전이와 만석 제약이 들어올 때마다 `u_day`의 분포를 갱신한다.

```text
p(u_day | 당일 관측)
∝ p(당일 관측 | u_day) · p(u_day | 날짜 조건)
```

만석 표본도 우도에 남아 있으므로 혼잡한 날이 계산에서 사라지지 않는다. 출력에는 `u_day`의 점추정이 아니라 분포 전체를 전달한다.

### 4.7 후보 정류장 도착 직전 좌석을 예측한다

현재 차량의 확정 가능한 최신 상태에서 후보 정류장 직전까지 각 정류장의 `N`을 순차적으로 생성한다.

```text
S⁻ᵣ,ₘ = propagate(S_current, Nᵣ,ₓ₊₁, …, Nᵣ,ₘ₋₁)
```

각 정류장의 시간 버킷은 현재 시각이 아니라 해당 차량의 예상 도착시각으로 선택한다. 차량이 이동 중 30분 버킷 경계를 넘으면 정류장마다 다른 버킷이 적용된다.

## 5. 층 2: 사용자 경쟁 수요 모형

### 5.1 직접 추정할 대상

개인 성공에 필요한 값은 전체 정류장 순수요 `N`이 아니라 다음 값이다.

```text
Hᵣ,ₘ = Qᵣ,ₘ(τᵘₘ) − Aᵣ,ₘ
```

`Q`는 사용자가 줄에 들어온 순간 앞에 있던 사람 수이고, `A`는 해당 차량에서 내린 사람 수다. 사용자 실측에서 두 값을 따로 기록하되 모델은 성공식에 직접 들어가는 `H`도 함께 학습한다.

### 5.2 필요한 직접 기록

개인 확률을 산출하려면 기준 정류장뿐 아니라 실제 추천 후보에서도 다음 기록이 필요하다.

- 사용자가 후보 정류장에 도착한 시각
- 도착 순간 앞에 있던 대기자 수
- 대상 버스의 실제 도착·승차 시작 시각
- 하차 인원
- 사용자 탑승 성공 여부
- 앞차 이후 경과시간 또는 실제 배차 간격
- 줄 순서가 무너졌는지 여부

탑승 후 API 차분으로 전체 탑승 인원까지 확인할 수 있으면 보조 진단에 사용한다. 그러나 전체 탑승 인원을 `Q`의 정답으로 사용하지 않는다.

### 5.3 경쟁 수요 모형

`H`도 노선 상태와 같은 일별 혼잡 효과를 공유한다.

```text
Hᵣ,ₘ = ηₘ,ᵦ,ᵍ
       + cₘu_day
       + θᵀx_day
       + φ₁·예상 선행시간
       + φ₂·headway
       + ζᵣ,ₘ
```

- `예상 선행시간 = Tᵇ−τᵘ`: 사용자가 버스보다 얼마나 먼저 도착하는지
- `headway`: 대기열 누적과 앞차 만석 이월을 설명하는 핵심 변수
- `cₘu_day`: 좌석 상태 모형과 공유하는 당일 혼잡 효과

선행시간이 길수록 항상 앞 대기자가 줄어드는 단순 선형 관계를 강제하지 않는다. 시간 구간별 효과나 단조 스플라인을 사용하고 실제 기록으로 형태를 검증한다.

### 5.4 직접 기록이 부족한 후보의 출력

직접 `Q`와 `A`를 학습할 자료가 없는 후보에는 개인 탑승확률 점추정을 내지 않는다. 대신 다음 사건을 계산한다.

```text
Rᵣ,ₘ = Fᵣ,ₘ · 1[S⁺ᵣ,ₘ > 0]
```

사용자가 승차 마감 전에 정류장에 있고 정류장 처리 후에도 좌석이 남았다면, 정상적인 승차 절차에서는 사용자가 탑승했음이 보장된다. 따라서 다음 하한이 성립한다.

```text
P(Yᵣ,ₘ=1) ≥ P(Rᵣ,ₘ=1)
```

`S⁺=0`이어도 사용자가 줄 앞쪽에서 실제로 탔을 수 있으므로 이 값은 보수적이다. 직접 기록이 쌓여 개인 모형의 예측구간이 충분히 좁아졌을 때만 하한 출력에서 개인 확률 출력으로 승격한다.

### 5.5 선착순 가정이 깨지는 경우

승차 위치가 여러 곳이거나 줄 순서가 자주 무너지면 `Q`만으로 탑승 순서를 설명할 수 없다. 이 경우 다음 중 하나가 필요하다.

- 실제 사용자 앞 탑승 인원을 별도로 기록한다.
- 줄 이탈·합류를 포함한 순서 오차항을 추가한다.
- 해당 정류장은 개인 확률 대신 보수적 하한만 제공한다.

선착순이 확인되지 않은 상태에서 정교한 확률을 출력하지 않는다.

## 6. 이동 가능성과 ETA 모형

후보 정류장까지의 사용자 이동 시간과 버스 도착 시간을 모두 분포로 추정한다.

```text
τᵘₘ = t₀ + T_user,m
Tᵇᵣ,ₘ = T_bus,r,m
Fᵣ,ₘ = 1[τᵘₘ + δₘ ≤ Tᵇᵣ,ₘ]
```

`T_user,m`은 정거장 수가 아니라 실제 이동 방법별 시간 분포다. 사용자의 과거 이동 기록이 없으면 지도 예상시간에 안전 여유를 크게 둔다. `T_bus`는 현재 차량 위치, 최근 구간 속도, 동일 시간대 구간 운행시간으로 예측한다.

다음 후보는 즉시 제외한다.

- 대상 차량이 이미 지나간 정류장
- 도착 가능성의 하한이 사용자 기준보다 낮은 정류장
- 이동 중 횡단·환승 등 실패 위험을 산출할 수 없는 정류장

버스 ETA와 좌석 상태는 완전히 독립적이지 않을 수 있다. 배차가 벌어진 차량은 더 긴 대기열을 만나 지연과 좌석 감소가 동시에 발생할 수 있으므로 `headway`와 운행 효과를 공유한다.

## 7. 공동 예측

후보별 확률을 따로 계산한 뒤 비교하지 않는다. 한 번의 시뮬레이션에서 모든 후보가 같은 날씨, 일별 혼잡도, 대상 차량 상태를 공유하게 한다.

각 시뮬레이션 경로에서 다음을 수행한다.

1. 모형 파라미터의 불확실성을 반영해 파라미터 한 세트를 뽑는다.
2. 현재까지의 당일 관측에 조건부인 `u_day`를 뽑는다.
3. 후보별 사용자 이동 시간과 대상 차량 ETA를 뽑는다.
4. 현재 좌석에서 후보 정류장 직전까지 `N`을 순차 생성해 `S⁻`를 얻는다.
5. 같은 `u_day`, 날씨, 배차 간격을 조건으로 후보별 `H`를 생성한다.
6. 각 후보의 `F`, `Y`, 이동 시간, 실패 손실을 계산한다.

이 과정을 반복하면 다음을 동시에 얻는다.

- 후보별 탑승 성공확률
- 후보별 이동 성공확률
- 후보별 실패 원인 분해: 늦게 도착 / 좌석 부족 / 불확실성 과다
- 기준 정류장 대비 개선확률
- 후보 간 기대손실 차이의 분포

공유 시나리오를 사용하기 때문에 비 오는 혼잡한 날에 모든 후보가 함께 나빠지는 상관관계와, 상류로 갈수록 좌석은 늘지만 도착 실패가 커지는 상충관계를 보존한다.

## 8. 추천 의사결정

### 8.1 기대손실

후보 행동 `a_m`의 예시 손실은 다음과 같다.

```text
L(aₘ) = w_move · T_user,m
       + w_wait · T_wait,m
       + w_fail · 1[Yᵣ,ₘ=0]
       + w_late · 1[목적지 도착 마감 초과]
```

가중치는 모델이 임의로 정하지 않는다. 사용자가 `탑승 실패를 피하기 위해 추가로 몇 분까지 이동할 수 있는지`를 설정하면 그 값에서 유도한다. 설정이 없으면 확률과 이동 시간을 함께 보여주고 단일 최적안을 강제하지 않는다.

### 8.2 추천 조건

평균 손실이 조금 낮다는 이유만으로 이동을 권하지 않는다. 기준 정류장 `M`과 후보 `m`을 같은 시나리오에서 비교한다.

```text
P(L(aₘ) + margin < L(a_M) | 현재 정보) ≥ q
```

- `margin`: 이동을 정당화할 최소 실익
- `q`: 추천 확신 기준

두 값은 백테스트와 사용자 위험 선호로 정한다. 예를 들어 `q=0.8`은 가능한 초기값일 뿐 통계적 진리가 아니다.

### 8.3 보류가 정상 출력이다

다음 상황에서는 추천을 보류한다.

- API 상태가 오래됐거나 운행 재구성 신뢰도가 낮음
- 후보 정류장의 개인 경쟁 수요 자료가 부족하고 보수적 하한도 기준을 넘지 못함
- 예측구간이 넓어 후보 간 우열이 불명확함
- 현재 조건이 학습 범위를 크게 벗어남
- 대상 차량보다 사용자가 먼저 도착할 가능성이 낮음

보류는 모델 실패가 아니라 거짓 정밀도를 방지하는 의사결정이다.

## 9. 불확실성 산출과 출력 규약

합성된 탑승 확률 전체에 Wilson 구간을 적용하지 않는다. 불확실성은 최소 네 종류를 포함한다.

1. 파라미터 불확실성: 작은 표본 때문에 `μ`, 일별 효과, 대기열 효과를 정확히 모르는 정도
2. 과정 불확실성: 같은 조건에서도 실제 승하차가 달라지는 정도
3. 관측 불확실성: API 지연·오류와 정류장 매핑 오류
4. 행동 불확실성: 사용자 이동 시간과 버스 ETA 오차

계층 베이지안 사후예측 또는 날짜·운행 단위 블록 부트스트랩으로 이 네 요소를 예측 경로에 전달한다. 같은 날짜의 정류장 관측을 개별 독립 표본으로 재표집하지 않는다.

Wilson 구간은 다음처럼 직접 관측한 이항 비율에만 제한적으로 사용한다.

- 특정 조건의 실제 만석 빈도
- 실제로 선택한 행동의 탑승 성공률
- 보정 구간별 경험 성공률

사용자 화면에는 후보별로 다음을 표시한다.

- 개인 탑승확률 또는 보수적 하한이라는 출력 종류
- 중심 추정치와 예측 불확실성 구간
- 제시간 도착확률
- 직접 실측 일수와 계층 풀링 비율
- 만석·장구간 관측이 차지하는 비율
- 데이터 최신성
- 추천 보류 또는 외삽 경고

고정된 `셀당 10회` 규칙은 폐기한다. 표본 수는 함께 보여주되 출력 가능 여부는 백테스트에서 정한 최대 허용 구간폭, 유효 표본 크기, 외삽 여부로 판단한다.

## 10. 데이터 설계

### 10.1 원시 관측

```text
(observed_at, api_timestamp, pseudonymous_vehicle_id,
 route_id, direction, seq, remaining_seats,
 collection_mode, page_active, response_age)
```

수집 방식과 응답 나이를 저장해야 기저 자료와 사용자 활성 자료의 선택 차이, API 지연을 추적할 수 있다.

### 10.2 운행 전이 자료

```text
(run_id, from_seq, to_seq, from_state, to_state,
 elapsed_time, mapping_confidence, transition_quality)
```

`transition_quality`는 정확한 단일 정류장 전이, 짧은 다중 전이, 장구간 전이, 상태 불명으로 구분한다. 품질이 다른 자료에 같은 가중치를 임의 부여하지 않고 각각에 맞는 관측 우도를 사용한다.

### 10.3 사용자 실측

```text
(decision_at, candidate_stop, user_arrived_at,
 target_run_id, bus_arrived_at, queue_ahead_at_arrival,
 alight_count, boarded, queue_order_broken,
 chosen_action, recommendation_shown)
```

추천 후보 전체의 개인 확률을 원하면 각 후보에 대한 실측이 필요하다. 기준 정류장 기록만으로 상류 후보의 대기열 분포까지 확인했다고 간주하지 않는다. 인접 정류장 풀링은 초기 사전분포를 제공할 뿐 직접 검증을 대체하지 못한다.

### 10.4 배차 간격과 이월 수요

동일 방향 차량들의 정류장 통과시각으로 실제 headway를 계산한다. 직전 차량이 만석이었다면 다음 차량의 대기열이 커질 수 있으므로 다음 변수를 둔다.

- 직전 차량 이후 경과시간
- 직전 차량의 잔여석 0 여부와 지속 구간
- 앞차와 뒷차의 간격 변화

v2.1에서는 이 값을 `H`의 설명변수로 사용한다. 미탑승 인원 수 자체를 복원하려 하지 않는다.

## 11. 검증 설계

### 11.1 수식과 구현 검증

실데이터 전에 합성 운행을 만들어 다음 상황을 재현한다.

- 비만석 단일 전이
- 만석 도달 후 계속 만석
- 중간 만석 후 하차로 좌석 회복
- 음의 순좌석 압력이 큰 정류장
- API 관측 누락과 지연
- 배차 간격 급증과 대기열 증가

알려진 잠재값으로 생성한 자료에서 상태모형이 검열 꼬리와 도착 좌석 분포를 회복하는지 확인한다. 이 검증을 통과하지 못하면 실데이터 결과가 그럴듯해 보여도 사용하지 않는다.

### 11.2 시간 순서 백테스트

무작위 행 분할을 사용하지 않고 날짜 단위 rolling-origin 평가를 사용한다.

```text
1~n일 학습 → n+1일 예측
1~n+1일 학습 → n+2일 예측
...
```

같은 날의 관측이 학습과 평가에 동시에 들어가 일별 효과를 미리 알게 되는 누수를 방지한다.

### 11.3 층별 평가 지표

**노선 좌석 상태 모형**

- 후보 정류장 도착 전 잔여석 MAE
- 잔여석 분포의 CRPS 또는 로그 점수
- 만석 사건 Brier score와 calibration
- 예측구간 실제 포함률
- 최초 소진 지점 오차

**사용자 경쟁 수요 모형**

- `H=Q−A`의 MAE와 예측구간 포함률
- 실제 탑승 성공의 Brier score와 log loss
- 확률 구간별 실제 탑승률
- 직접 관측 후보와 풀링 후보의 성능 차이

**ETA·이동 모형**

- 버스 도착시각 절대오차
- 사용자 이동시간 절대오차
- `제시간 도착` 사건의 calibration

### 11.4 기준선

복잡한 모형은 반드시 다음보다 나아야 한다.

1. 현재 잔여석만 사용하는 규칙
2. 시간대별 과거 만석 빈도
3. 사용자의 기존 경험칙
4. `정류장 처리 후 잔여석 > 0`만 예측하는 보수적 모형

복잡한 모형이 단순 기준선보다 Brier score, 실패 비용 또는 calibration에서 안정적으로 낫지 않으면 단순 모형을 유지한다. 기준선 2는 v1 명세(`docs/boarding-model.md`)의 만석 빈도 확률이다.

### 11.5 선택하지 않은 후보의 반사실을 만들지 않는다

사용자가 기준 정류장에 남았다면 상류 정류장에서 실제로 탔을지는 관측되지 않는다. 같은 시간대 API 스냅샷만으로 개인의 도착 가능성과 대기열 순서를 알 수 없으므로 이를 성공·실패 정답으로 사용하지 않는다.

정책 자체의 우수성을 비교하려면 다음 중 하나가 필요하다.

- 안전한 후보들이 비슷한 날에 제한적으로 무작위 선택하고 선택확률을 기록한다.
- 장기간 실제 선택된 행동의 calibration만 평가하고 대안 정책의 우월성 주장을 보류한다.
- 별도 관찰자가 여러 후보 정류장의 대기열과 하차를 같은 운행에서 기록한다.

출근 지각 비용이 큰 상황에서 무리한 탐색을 강제하지 않는다. 데이터가 부족하면 정책 우월성보다 선택된 행동의 확률 보정을 먼저 검증한다.

### 11.6 패턴 존재성 판단

평일 5일의 `±20%p` 반복 여부 같은 임의 규칙은 사용하지 않는다. 다음 질문으로 대체한다.

- 날짜를 바꿔 예측해도 단순 기준선보다 나은가?
- 날짜 간 분산이 날짜 내 분산보다 얼마나 큰가?
- 예측구간이 실제 빈도를 약속한 비율로 포함하는가?
- 비, 공휴일 등 조건이 바뀌어도 calibration이 유지되는가?

패턴이 존재한다는 주장은 표본 모양이 비슷해 보이는지가 아니라 미래 날짜 예측력이 반복되는지로 판단한다.

## 13. 기대되는 장점

### 13.1 논리적 일관성이 높아진다

관측된 정류장 전체 순변화와 개인 앞 대기열을 같은 변수로 취급하지 않는다. 입력과 출력 사이에 어떤 추가 정보와 가정이 필요한지가 명시된다.

### 13.2 혼잡한 날을 학습에서 지우지 않는다

만석 관측을 꼬리 제약으로 남기므로 실제로 중요한 고수요 상황이 평균에서 사라지는 현상이 줄어든다.

### 13.3 실제 행동 가능성을 평가한다

상류 정류장의 좌석만 보는 것이 아니라 사용자가 먼저 도착할 수 있는지까지 평가한다. 좌석은 많지만 갈 수 없는 후보가 추천되는 오류를 막는다.

### 13.4 후보 비교가 더 현실적이다

날씨와 일별 혼잡도를 공유해 모든 후보가 함께 나빠지는 날을 표현한다. 상류 이동의 좌석 이득과 이동 실패 위험을 같은 시나리오에서 비교할 수 있다.

### 13.5 거짓 정밀도를 줄인다

직접 자료가 부족하면 보수적 하한이나 보류를 출력한다. 숫자가 있다는 이유만으로 정확해 보이는 문제를 줄인다.

### 13.6 단계적으로 확장할 수 있다

노선 상태 모형만으로도 정류장 처리 후 좌석이 남을 확률을 제공할 수 있다. 이후 사용자 기록이 쌓이면 개인 경쟁 수요 층을 붙여 개인 확률로 승격할 수 있다.

### 13.7 실패 원인을 설명할 수 있다

추천 실패를 `ETA 실패`, `좌석 상태 오판`, `대기열 오판`, `API 지연`으로 나눌 수 있어 모델 개선 방향이 분명해진다.

## 14. 기대되는 단점과 남는 위험

### 14.1 추가 실측 없이는 개인 확률을 완성할 수 없다

가장 큰 단점이다. 기준 정류장뿐 아니라 후보 정류장의 사용자 도착 시점 대기열과 하차 인원을 기록해야 한다. 이 부담을 감수하지 않으면 보수적 하한까지만 제공할 수 있다.

### 14.2 구현 복잡도가 크게 증가한다

균등 배분과 평균 누적은 단순하지만 상태공간 추론, 계층적 효과, 공동 시뮬레이션은 구현·검증·운영이 어렵다. 오류가 나도 단순 평균보다 원인을 찾기 어렵다.

### 14.3 초기 데이터 구간에서 추천이 자주 보류된다

정직한 불확실성 기준을 적용하면 초기에 확률을 내지 못하는 후보가 많다. 제품의 즉각적인 체감 가치가 낮아질 수 있다.

### 14.4 모형 가정은 여전히 남는다

선착순 승차, 일별 효과의 공유 구조, 인접 시간대 부분 풀링, ETA 분포 같은 가정이 필요하다. 재설계는 가정을 없애는 것이 아니라 검증 가능한 형태로 드러낸다.

### 14.5 API 의미와 품질이 불명확하면 전체 모델이 무너진다

잔여석이 정류장 처리 전인지 후인지, 위치 정보가 얼마나 늦는지 확인되지 않으면 상태 전이가 잘못 정렬된다. 통계 모형을 정교하게 만들어도 원천 상태 정의가 틀리면 소용없다.

### 14.6 후보별 데이터 편향이 생긴다

사용자는 위험해 보이는 날에만 앱을 열고, 추천받아 이동한 정류장에서만 기록할 가능성이 높다. 수집 정책과 선택확률을 기록하지 않으면 모델이 자기 추천으로 만들어진 데이터에 편향될 수 있다.

### 14.7 극단 상황은 계속 어렵다

행사, 사고, 노선 우회, 장시간 배차 붕괴처럼 학습 범위를 벗어난 상황에서는 예측보다 보류가 맞다. 이런 날까지 정확한 확률을 보장할 수는 없다.

### 14.8 계산비용과 운영비용이 늘어난다

실시간 사후 갱신과 다수 시뮬레이션은 단순 합산보다 서버 자원을 더 쓴다. API 쿼터뿐 아니라 모델 계산 지연과 캐시 전략도 운영 제약이 된다.

### 14.9 정책 우월성을 입증하는 데 시간이 오래 걸린다

한 사용자는 동시에 여러 정류장을 선택할 수 없다. 선택하지 않은 후보의 실제 결과가 없으므로 추천 정책이 경험칙보다 낫다는 결론은 장기간의 실제 선택 자료 없이는 강하게 주장하기 어렵다.

---

## 15. 이 모델로도 알 수 없는 것

다음 항목은 추가 센서나 직접 기록 없이는 정확히 복원할 수 없다.

- 만석으로 인해 타지 못한 사람의 정확한 수
- 잔여석 차분만으로 분리한 정확한 탑승 인원과 하차 인원
- 사용자가 도착하기 전과 후에 온 사람의 구분
- 선택하지 않은 정류장에서 사용자가 실제로 탔을 반사실
- API에 반영되지 않은 현금 승차나 상태 오류의 정확한 크기

이 항목들을 숨은 변수로 둘 수는 있지만, 사전가정에 따라 결과가 달라진다. 민감도 분석에서 가정을 바꿨을 때 추천이 뒤집히면 확정 추천을 하지 않는다.

---

## 16. 구현 순서

### Phase 0. 원천 의미 검증

- API 잔여석과 위치의 시점 의미를 현장 기록과 대조한다.
- 차량별 좌석 정원과 운행 분할 규칙을 확인한다.
- API 지연·중복·역행·결측 패턴을 측정한다.

통과 조건: 정류장 처리 전후 상태를 재현 가능한 규칙으로 매핑할 수 있어야 한다.

### Phase 1. 좌석 상태 모형과 보수적 하한

- 정밀 전이와 장구간 전이를 분리한다.
- 검열 상태 전이를 구현한다.
- 현재 차량에서 후보 정류장까지 `S⁻`, `S⁺` 분포를 예측한다.
- 제시간 도착확률과 `P(F=1, S⁺>0)`를 출력한다.

이 단계만으로도 개인 확률이라고 과장하지 않는 실용적인 MVP가 된다.

### Phase 2. 사용자 경쟁 수요 모형

- 후보별 `Q`, `A`, 실제 결과를 수집한다.
- 선행시간과 headway 조건의 `H=Q−A` 분포를 학습한다.
- 직접 자료가 충분한 후보만 개인 확률로 승격한다.

### Phase 3. 공동 예측과 의사결정

- 좌석 상태와 `H`가 일별 효과를 공유하도록 결합한다.
- 후보별 이동·ETA를 같은 시나리오에서 시뮬레이션한다.
- 기대손실과 기준 정류장 대비 개선확률로 추천한다.

### Phase 4. 보정과 정책 평가

- 날짜 순서 백테스트와 실제 사용 calibration을 누적한다.
- 단순 기준선보다 개선되지 않는 구성요소를 제거한다.
- 충분한 근거가 있을 때만 다음 차량과 이월 대기열 모형으로 확장한다.

---

## 17. 최종 판단

재설계의 가장 중요한 변화는 복잡한 통계기법을 추가한 것이 아니다. **잔여석으로 알 수 있는 것과 개인 기록이 있어야 알 수 있는 것을 분리한 것**이다.

이 설계는 기존안보다 구현이 어렵고, 초기에 보여줄 수 있는 확률도 적다. 반면 다음 세 가지 주장은 논리적으로 방어할 수 있다.

1. 만석과 장구간 관측을 포함해 현재 차량의 후보 정류장 도착 좌석을 예측한다.
2. 사용자의 도착 시각과 직접 관측한 앞 대기열을 결합해 개인 탑승 성공을 추정한다.
3. 자료가 부족한 경우 개인 확률 대신 보수적 하한을 제시하고 추천을 보류한다.

개인 대기열 실측을 후보 정류장까지 확장할 의사가 없다면 Phase 1에서 멈추는 것이 가장 정직하고 비용 대비 효과도 좋다. 반대로 실측 부담을 감수할 수 있다면 Phase 2 이후부터 비로소 `개인 탑승 성공확률`이라는 표현을 사용할 근거가 생긴다.

## 부록 A. 채택 근거 검증 기록 (2026-07-21)

v1에서 v2로 방향을 바꾼 근거를 세 층으로 구분해 기록한다. 자료는 2026-07-19(일)부터 2026-07-21(월) 오전까지의 API 관측 3일치이며, Phase 0(현장 대조) 이전이므로 모든 수치는 API 관측 기준이다.

### A.1 수학적으로 확정된 것

- 잔여석 차분의 개인 탑승확률 식별 불가능성 (1.1절). 반례 검산으로 확인했다. `S⁻=3, S⁺=1` 관측이 (A=0, D=2)와 (A=5, D=7) 모두와 양립하고, 사용자 앞 4명 조건에서 두 상황의 개인 결과가 갈린다. 표본 증가로 해소되지 않는다.
- `D`와 `Q`의 개념 구분 (1.2절). `D ≥ Q`이므로 대용 사용의 국소 효과는 경쟁 과대평가다. 단 v1 전체의 순편향 방향은 반대 방향 효과(만석 제외, 숨은 검열)와 섞여 자료 없이 확정할 수 없다.

### A.2 3일 API 자료에서 확인된 강한 신호

| 관측 | 수치 | 해석 범위 |
|---|---|---|
| 검열(끝점 0 포함) 관측쌍의 시간대 분포 | 아침 피크 15.0%, 저녁 피크 7.6%, 그 외 2.0% | 선택적 결측의 존재는 확인. 같은 셀 내부의 편향 크기는 미측정 |
| 만석 후 회복 전이 (0석에서 양수로) | 61건, 차량 52대와 차량×날짜 60조합에 분산 | 회복 동역학의 존재와 비군집성 확인. 특정 차량·날짜 반복 관측 아님 |
| 음수 순수요 셀 비중 | 유효 셀(가중치 0.5 이상)의 52~54% | 부호 있는 `N`의 실재 확인. 실제 λ 계산의 불안정성은 미계산 |
| 중복 의심 관측 (동일 위치·좌석, 5분 미만) | 전체의 3.5% (상한) | 집계 왜곡 요인으로는 작음 |

### A.3 직접 검증으로 승격된 것

- **양수→0→양수 삼중 관측 11건.** 양수 끝점 사이에 만석 상태가 실제로 낀다는 직접 증거다. 예: 3330, 2026-07-21 07시, seq21(9석) → seq25(0석) → seq28(3석). 만석 후 회복을 허용하는 상태 전이(4.3절)는 필수로 확정된다.
- **다운샘플링 테스트: 숨은 검열률 0.9%.** 관측 하나를 건너뛴 양수-양수 쌍 1,213건 중 건너뛴 중간 관측이 0석인 경우가 11건이었다. 해상도가 절반(약 20분)이었다면 이 비율만큼이 깨끗한 쌍으로 위장된다. 현상은 실재하되 현재 10분 해상도에서의 크기는 약 1%로 작다. 이전 보고에서 제시한 32.4%는 위험 노출 상한이지 숨은 검열률이 아니다. 10분보다 짧은 만석 구간은 이 방법으로 측정되지 않으므로, 30초 라이브 자료 축적 후 재측정한다.

### A.4 아직 입증되지 않은 것

- v1 각 편향의 크기와 순방향
- v2 구성요소들의 예측 우월성 (11.4절 기준선 대비 백테스트 전)
- 실제 계산된 λ의 불안정성 (분모 크기, leave-one-out 민감도 미계산)
- API 값과 실제 승하차 상태의 일치 (Phase 0 전)

### A.5 결론 문구

v1의 개인 탑승확률에는 구조적 식별 문제가 있으며, 이는 수학적으로 확정된다. 3일간의 API 자료에서는 만석 후 회복, 피크 시간대 검열 집중, 다수의 음수 순수요 셀이 관측되어 해당 문제가 실제 운영 자료에서도 무시하기 어렵다는 강한 신호를 확인했다. 다만 현재 분석은 편향의 정확한 크기나 v2의 예측 우월성을 입증한 것은 아니다. v2를 공식적인 설계 방향으로 채택하되, API 의미 검증과 단순 기준선 대비 백테스트를 통과하기 전까지는 검증 중인 명세로 둔다.
